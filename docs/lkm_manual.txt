Sendo aprovado nesta primeira validação, a segunda validação ocorrerá. Validação da Assinatura Digital: O Validador faz uma validação completa na Assinatura digital, verificando se os dados do XML foram alterados e também se a assinatura está dentro dos padrões técnicos exigidos pela SEFAZ e pelas as autoridades certificadoras. Validação junto à SEFAZ (verifica a situação da NF-e/CT-e em questão) : Após garantir que o arquivo XML é íntegro, o Validador consulta a Chave de acesso do XML na SEFAZ para verificar o status da NF-e/CT-e (Aprovada, Cancelada, Rejeitada , inexistente, erro, etc...) e retorna o status ao SAP. Nota : Independentemente de qual seja o retorno obtido, o status será sempre repassado ao SAP, finalizando assim o processo de validação deste XML. 4. 1.2 Manifestação do Destinatário O Validador e stá habilitado a realizar as operações de Manifestação do Destinatário, conforme constam na Nota Técnica disponibilizada pela SEFAZ: Eventos NF-e 210200 - Confirmação da Operação 210210 - Ciência da Operação 210220 - Desconhecimento da Operação 210240 - Operação não Realizada Evento CT-e Prestação do Serviço em Desacordo 4. 1. 3 Serviços de Distribuição de DF-e O Validador também utiliza o “Serviço de Consulta da Relação de Documentos Destinados” para verificar as NF-es emitidas contra o CNPJ cadastrado no SAP (e suas respectivas Cartas de Correção Eletrônicas (CC-e) e cancelamentos). Além disso, há a possibilidade de utilizar o “Serviço de Download da NF-e” informando a chave de acesso para obtenção do XML desejado, de acordo com as regras estabelecidas pela SEFAZ . 4.2 Extrator O extrator fica monitorando a caixa de e-mail em busca de arquivos a serem processados, a aplicação realiza o download e o processamento do arquivo. Métodos de acesso suportados pela a aplicação: POP3: (Post Office Protocol versão 3) é usado para se comunicar com o servidor de e-mail e baixar os e-mails para um cliente de e-mail IMAP: (Internet Message Access Protocol ), como o POP3, também é usado para recuperar e-mails para empresas de e-mail API de E-mail do Outook do Office 365: A API do E-mail do Outlook permite ler, criar e enviar mensagens e anexos, visualizar e responder a mensagens de eventos e gerenciar pastas protegidas pelo Active Directory do Azure no Office 365. 4.2.1 Extração de XML do e-mail O Extrator de XML de e-mail realiza a extração de XMLs e trata-os de três maneiras distintas: Salvá-los em um ou mais diretórios pré-definidos em arquivo de configuração do produto. Reenviá-los para outra conta de e-mail. Enviá-los diretamente ao SAP através de chamadas de funções. A utilização do Extrator pode ser feita de duas formas: Pode-se extrair XMLs provenientes dos fornecedores, ou seja, XMLs de recebimento (NF-e de entrada) para posterior armazenamento, bastando para isso apenas configurar a conta de e-mail de recebimento destes arquivos e definir o tratamento a ser realizado com o XML. A mesma lógica pode ser utilizada parar os XMLs enviados. Há casos em que a empresa opta que para cada NF-e que ela emita, uma cópia do arquivo XML seja enviada para um e-mail da própria empresa para controle e redundância de segurança. Pode-se configurar a conta de e-mail que irá receber esses arquivos da mesma forma citada anteriormente. Nota : O Extrator não faz qualquer alteração ou validação nos XMLs extraídos dos e-mails. 4.2.2 Extração de PDF/IMAGEM do e-mail A solução utiliza o software OCR integrado ao AR-DFE para realizar a transformação de documentos eletrônicos no formato de imagem PDF em arquivo TXT, após transformar o arquivo é disponibilizado em diretório para obtenção do SAP ECC. Resumidamente o OCR funciona de acordo com o macro-fluxo : Todos os métodos do webservice geram informações de acompanhamento que são armazenadas no banco de dados, para controle da solicitação\arquivo e seu fluxo. Essas informações são armazenadas em forma de fluxo, permitindo obter informações detalhas sobre um determinado documento ou gerar estatísticas de utilização do processo por um determinado cliente, e em forma de log histórico, detalhando o a execução de um método de forma detalhada, permitindo a verificação de uma inconsistência, caso necessário. Detalhamento do processamento: Prestador emite NFS-e/ DOC-e , disponibiliza PDF no e-mail ou Usuário encaminha PDF no e-mail. Extrator (JAVA) obtém PDF do servidor de e-mail. Extrator grava em diretório s A (Interno) e-mail + anexo (Parâmetros 2.1 item 1). JOB SAP (Parâmetros 2.1 item 2 (PGM 30) chama webservice que obtém PDF no diretório A (Interno) local e grava no diretório B (LKM). OCR obtém PDF no diretório B (LKM) transforma em TXT e grava no diretório C (LKM). JOB SAP (Parâmetros 2.1 item 3) (PGM 29) chama webservice que obtém TXT + PDF (Anexos) no diretório C (LKM) e importa no SAP. 4.2.3 Apaga os e-mails já processados da conta O Extrator extrai os XMLs e apaga as mensagens do servidor de e-mail, evitando assim o grande acumulo de e-mails e sobrecarga de dados na caixa do correio eletrônico. 4.2.4 Extração de arquivos em formato diferente de XML O Extrator extrai todos os arquivos a nexados ao e-mail sendo estes XML ou não e, realiza tratamento diferenciado para os arquivos que não são XML. Estes podem ser reenviados para outra conta de e-mail ou armazenados em diretório. 4.2.5 Comunicação com o SAP A partir de uma configuração no arquivo de configuração “ configuration.properties ” é possível estabelecer comunicação entre o SAP e o Extrator, que envia o XML para o ERP após extraí-lo do e-mail. N ota : O Extrator de e -mail foi desenvolvido para se comunicar com SAP, todavia as funções e demais objetos do ERP não estão contempladas pelo produto Extrator de e -mail. 4.2.6 Redirecionamento de e-mails Qualquer e-mail recebido dos fornecedores, que esteja vazio ou que contenha anexos diferentes de XML, será redirecionado para outra conta de e-mail (parametrizável no arquivo configuration.properties ) para que seja analisado a fim de definir o que vai ser feito com o mesmo. Estes e-mails redirecionados serão apagados da caixa de entrada em que foram recebidos. Esta regra vale tanto para e-mails que só contenham arquivos diferentes de .XML, assim como e-mails que contenham arquivos .XML e outros arquivos diferentes de .XML. No caso desta funcionalidade não ser utilizada, o comportamento continua ser de excluir o e-mail da caixa de entrada. Nota : A conta utilizada para enviar os e-mails é a exatamente a mesma que o Extrator acessa para baixar os XMLs recebidos dos fornecedores. 4.2.7 Tratamento para arquivos diferentes de XML Existe no arquivo de configuração um parâmetro para definir se arquivos diferentes de XML serão salvos e outro para indicar o diretório onde estes arquivos serão baixados. A partir disto, são adotadas as seguintes regras: E-mail sem nenhum anexo: O e-mail será excluído da caixa de entrada. E-mails somente com o XML: O XML será extraído para a pasta da rede/servidor e o e-mail será excluído da caixa de entrada. E-mails com XMLs e outros arquivos anexados: O XML será extraído para a pasta de rede de XML, os outros arquivos anexos serão extraídos para outra pasta de rede de outros arquivos que será definida e o e-mail excluído da caixa de entrada. E-mails com anexos diferente s de XML (.PDF, .DOC, .TXT, outros): Os arquivos anexos serão extraídos para outra pasta de rede de outros arquivos e o e-mail excluído da caixa de entrada. 5 INSTALAÇÃO DAS APLICAÇÕES 5.1 Instalação do JRE/JDK A aplicação utiliza o JRE/JDK da Amazon Correto para executar os processos. Para gerá-la, basta executar a biblioteca Cifrador.jar, localizada no diretório lib. A ex ecução, em ambiente Windows, é realizada através de um duplo clique na própria biblioteca. Ao iniciar, a seguinte tela é apresentada: No campo “Senha”, informe a senha do usuário. No campo “Confirma senha”, informe novamente a senha do usuário. O botão “Exibir senha” permite visualizar as senhas informadas Se as senhas estiverem iguais, a senha criptografada é apresentada no campo “Senha criptografada”. O botão “Copiar para a área de transferência” disponibiliza a senha criptografada no “clipboard” do Windows para permitir que seja colada no arquivo de configuração. 5.4 Configuração de Integração via Cloud Integration Pré-requisitos: Ter o Fluxo de Integração configurado no SAP BTP 1 – Configuration connection BTP: Open the ../bin/ btp.properties file clientId : clientId do BTP clientSecret : clientSecret do BTP tokenUrl : URL que gera o TOKEN de autenticação da aplicação que está hospedada no BTP webServiceUrl : URL do aplicativo que está hospedado no BTP 2 - Habilitando a integração por BTP Para Extrator: Abra o arquivo ../bin/ configuration.properties Para Validador: Abra o arquivo ../bin/ configsap.properties Defina o parâmetro usaBtp como true Exemplo: usaBtp = true
